# 多模态对话中的情感原因解释任务说明

## 背景

多模态对话中的情感理解不仅依赖文本内容，还与说话人的语气、视频画面、面部表情以及上下文互动过程密切相关。在真实对话场景中，人物情绪往往并不是由单一句子直接决定，而是由多轮对话逐步形成，并通过语言、语音和视觉信号共同表现出来。

已有情感原因研究大多采用抽取式范式，即从历史话语中定位某一句或某几句作为情绪原因。这类方法便于建模和评测，但通常只能给出较粗粒度的答案，难以覆盖更隐含、更复杂的情绪触发机制。

针对这一问题，本文聚焦多模态对话情感原因解释任务，即 **Multimodal Emotion Cause Explanation in Conversation (MECEC)**。该任务要求模型在理解对话语境和多模态证据的基础上，生成目标话语情绪成因的自然语言解释，从而更完整地揭示情绪产生的原因。

## 测评内容

MECEC 任务关注目标话语情绪成因的解释生成。给定一段多模态对话上下文、目标话语及其对应的视频和辅助转写信息，模型需要输出与该目标情绪相关的原因说明。

该任务主要覆盖以下几个方面：

- 对话历史中关键触发因素的定位；
- 文本、语音、视频等多模态证据的联合建模；
- 自然语言形式的情感原因解释生成。

与传统情感原因抽取任务相比，该任务更强调解释的完整性、可读性和细粒度表达能力，而不是只给出某个原因片段的位置。

## 数据集

该任务的数据构建基于多模态情绪对话语料，典型数据来源建立在 **MELD** 数据集基础之上，并进一步扩展为适用于情感原因解释生成的标注形式。数据集的主要特点如下：

- 包含 **7,273** 个视频片段；
- 保留文本、音频、视频三种模态信息；
- 去除 `neutral` 类别，仅保留六类离散情绪；
- 每个样本配有人工编写的情感原因解释。

数据中涉及的六类情绪包括：

- `joy`
- `sadness`
- `anger`
- `fear`
- `disgust`
- `surprise`

## 测评任务

MECEC 的核心任务是情感原因解释生成。给定目标话语及其多模态上下文，模型需要围绕目标话语所对应的情绪状态，生成完整、合理且具有可解释性的原因描述。该任务重点不在于预测情绪类别本身，而在于解释情绪产生的原因。

### 示例

```text
Input:
目标语句：Ross: Oh Mon, I laughed so hard
目标语句情感：joy
辅助信息：包含视频片段和历史上下文

Output:
说话人之所以表现出愉快情绪，是因为他得知 Chandler 误亲了一名男子，这一出乎意料又带有喜剧效果的事件让他觉得非常好笑，因此产生了明显的开心和兴奋反应。
```

### 评价重点

- 是否准确解释目标情绪的成因；
- 是否综合利用多模态证据；
- 解释内容是否连贯、清晰、具备可读性；
- 是否能够表达隐式因果关系。

## 评价指标

该任务属于生成式解释任务，因此评价时需要同时考虑生成文本与参考解释之间的内容一致性和语言质量。当前文档仅保留以下两个测评指标：

- **ROUGE**：用于衡量生成结果与参考解释在召回层面的重合程度；
- **BERTScore**：基于上下文语义表示，计算生成解释与参考解释之间的语义相似度。

## 输入与输出

### 输入形式

模型输入通常包括以下内容：

- 目标话语；
- 目标话语情感；
- 对话历史；
- 视频片段或视频特征。

### 输出形式

模型输出为关于目标情绪成因的自然语言解释，重点是生成完整、清晰且具有可读性的原因说明文本。

## 任务难点

MECEC 任务的主要挑战包括：

- 多模态证据分散在文本、语音和视觉信号中，信息融合难度较高；
- 情绪原因常常是隐式的，需要结合上下文推断；
- 对话中的情绪变化具有明显上下文依赖，建模跨度较长；
- 生成结果不仅要正确，还要自然、清晰并具备解释性。

## 任务意义

MECEC 比传统抽取式情感原因任务更接近真实的人类理解过程。它重点关注“人物为什么会产生这种情绪”的解释生成。因此，该任务对于以下研究方向具有重要意义：

- 多模态情感理解；
- 可解释情感分析；
- 对话系统中的情绪感知与响应；
- 更具共情能力的人机交互系统。

## 总结

MECEC 是一个面向多模态对话理解的生成式任务，核心目标是在上下文和多模态证据支持下，对目标话语的情绪成因给出自然语言解释。整体上，它强调原因解释生成，是多模态情感理解与可解释建模的重要研究方向。
